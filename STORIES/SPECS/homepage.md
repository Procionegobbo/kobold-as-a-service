# Homepage — Implementation Specification

## Feature Name & Description

Replace the default Laravel welcome view with the Kobold As A Service (KAAS) branded homepage: a single, server-rendered marketing page that explains the project, showcases live API usage examples in multiple languages, displays a sample generated kobold, links to related open-source projects, and includes a branded footer with attribution.

**Current state.** `resources/views/welcome.blade.php` contains the stock Laravel skeleton welcome screen. The `GET /` route returns that view unchanged. No KAAS branding, documentation, or code examples exist.

**In scope.**
- Replace `welcome.blade.php` with the full KAAS homepage (same file).
- Rename the view to `home` and update the web route accordingly, or keep the filename `welcome` — see decision below.
- All content sections described in the draft.
- Responsive layout using Tailwind CSS v4 (already installed).
- No new JS framework; plain vanilla JS for the language tab switcher only.
- A named web route `home` pointing to `GET /`.

**Out of scope.**
- Authentication pages or flows.
- Interactive live API calls from the browser (fetch to the API on the homepage — see Future Considerations).
- Any backend persistence, caching layer, or database changes.
- i18n/localisation of the homepage itself.

> **Note:** The output example card is generated **server-side on every request** via `KoboldGeneratorService`. This is server-rendered, not a browser-initiated API call.

---

## Architecture / Design Overview

The homepage is a pure server-rendered Blade view. There is no controller class needed; the existing inline closure in `routes/web.php` is upgraded to a dedicated `HomepageController` following the same pattern as `KoboldGeneratorController`. The view is `resources/views/home.blade.php`.

```
GET /
  └── HomepageController@__invoke
        ├── Cache::remember('homepage.kobold', 15s)
        │     └── KoboldGeneratorService::generate('en')  → $kobold (array)
        └── view('home', ['kobold' => $kobold])
              ├── resources/views/home.blade.php
              └── @vite(['resources/css/app.css', 'resources/js/app.js'])
```

The page is server-rendered on every request. All code snippets are hardcoded strings in the view. The output example card is generated by `KoboldGeneratorService::generate('en')` and cached in memory for 15 seconds under the key `homepage.kobold` — the grammar engine is invoked at most once per 15-second window.

**Key design decisions.**

| Decision | Choice | Reason |
|---|---|---|
| Controller vs. closure | `HomepageController` (invokable) | Consistent with existing `KoboldGeneratorController`; easier to test with `$this->get(route('home'))` |
| View filename | `home.blade.php` | Communicates intent; `welcome.blade.php` is framework boilerplate |
| Live vs. static output example | Live via `KoboldGeneratorService`, cached 15 s | Real output; grammar only runs once per 15-second window |
| Cache driver | In-memory (array driver in tests; default app cache in production) | No disk I/O; TTL short enough not to need invalidation logic |
| Tab switcher for code examples | Vanilla JS | No front-end framework installed; keeps bundle tiny |
| Dark mode | Supported via Tailwind `dark:` variants | The existing welcome view already carries full dark-mode styling |

**Page sections (top to bottom).**

```
[Header / Nav bar]          logo + project name
[Hero]                      headline + one-liner description
[API Code Examples]         tabbed code block (PHP, JS, Go, Python, cURL)
[Output Example]            rendered JSON card
[Inspirations & Credits]    Polygen + FumbleGDR acknowledgement
[Links]                     polygen-php on GitHub/Packagist, Laravel, procionegobbo.it
[Footer]                    "Made with ❤ by Federico 'Procionegobbo' Maiorini"
```

## Configuration

No new environment variables or feature flags are required.

The app name displayed in the `<title>` tag and the navigation already reads from `config('app.name')`. The `.env` value `APP_NAME` should be set to `Kobold As A Service` (this is an environment concern, not a code change — document it in `.env.example` with a comment).

**Cache driver.** The controller uses `Cache::remember('homepage.kobold', 15, …)`. Any driver that persists between requests works: `file` (Laravel default), `redis`, or `apc`. The `array` driver is per-request only and must **not** be used in production — it would bypass the 15-second cache entirely. In the test suite the `array` driver is fine because each test is a single request.

## Controller

File: `app/Http/Controllers/HomepageController.php`

Create using:
```bash
php artisan make:controller HomepageController --invokable --no-interaction
```

Resulting class:

```php
<?php

namespace App\Http\Controllers;

use App\Services\KoboldGeneratorService;
use Illuminate\Support\Facades\Cache;
use Illuminate\View\View;

class HomepageController extends Controller
{
    public function __invoke(KoboldGeneratorService $service): View
    {
        $kobold = Cache::remember(
            'homepage.kobold',
            15,
            fn () => $service->generate('en'),
        );

        return view('home', compact('kobold'));
    }
}
```

The controller resolves the kobold from the cache. On a cache miss it calls `KoboldGeneratorService::generate('en')`, stores the result under `homepage.kobold` for 15 seconds, then returns it. Subsequent requests within that window skip the grammar engine entirely. Laravel's service container auto-resolves `KoboldGeneratorService`.

### Header / Nav

A slim top bar with the project name "Kobold As A Service" on the left and a link to the API endpoint documentation anchor (`#api`) on the right. No authentication nav items — this project has no auth routes.

### Hero section

- `<h1>` — "Kobold As A Service"
- `<p>` — Short value proposition, e.g.: "Generate procedurally-created Kobold RPG character sheets via a simple HTTP API, powered by Polygen grammars."
- A CTA button linking to `#api` ("Try the API") and a secondary link to `#links` ("Explore the ecosystem").

### API Code Examples section

A tabbed interface with five language tabs: **PHP**, **JavaScript**, **Go**, **Python**, **cURL**.

Each tab shows a `<pre><code>` block with a realistic usage snippet for `POST /api/generate-kobold`. Tab switching is handled by a small inline `<script>` block at the bottom of the section (no external library). The active tab's code block is visible; others are `hidden`. The active tab button carries a distinct style (e.g., bottom border or background).

The hardcoded endpoint in all examples must be `POST /api/generate-kobold` with an optional JSON body `{"language": "en"}`.

**PHP example (using Guzzle):**
```php
$client = new \GuzzleHttp\Client();
$response = $client->post('https://kaas.procionegobbo.it/api/generate-kobold', [
    'json' => ['language' => 'en'],
]);
$kobold = json_decode($response->getBody(), true);
```

**JavaScript example (fetch):**
```js
const response = await fetch('/api/generate-kobold', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ language: 'en' }),
});
const kobold = await response.json();
```

**Go example:**
```go
body := strings.NewReader(`{"language":"en"}`)
resp, err := http.Post("https://kaas.procionegobbo.it/api/generate-kobold",
    "application/json", body)
// handle err, decode resp.Body into struct
```

**Python example (requests):**
```python
import requests
r = requests.post(
    "https://kaas.procionegobbo.it/api/generate-kobold",
    json={"language": "en"},
)
kobold = r.json()
```

**cURL example:**
```bash
curl -X POST https://kaas.procionegobbo.it/api/generate-kobold \
     -H "Content-Type: application/json" \
     -d '{"language": "en"}'
```

All five blocks must be present in the DOM at load time; only the visibility is toggled. This avoids any JS framework dependency and keeps the HTML accessible.

### Output Example section

Display a real kobold JSON output, freshly generated by `KoboldGeneratorService` on each page load and passed to the view as `$kobold`. Style it as a card or highlighted code block.

The view renders the `$kobold` array as a JSON-formatted block:

```blade
<pre><code>{{ json_encode($kobold, JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE) }}</code></pre>
```

The card should carry a heading like "Sample Output" and a brief sentence explaining these fields are generated by the Polygen grammar engine. Because the output is live, the displayed kobold will be different on every page load.

## Validation Rules

This feature introduces no user input on the homepage itself. No form requests are needed.

## Authorization & Security

The `GET /` route is public and unauthenticated. No policy, middleware, or guard changes are needed. Because there is no form submission, no CSRF token is required on this page.

All external links must include `rel="noopener noreferrer"` to prevent reverse tabnapping.

No rate limiting is required for a static GET route (the existing global throttle middleware applied to API routes does not affect web routes).
