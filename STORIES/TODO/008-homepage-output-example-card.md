# 008 - Homepage Live Output Example Card

## Story

**As a** visitor curious about what KAAS produces,
**I want** to see a real, freshly generated kobold character displayed on the homepage,
**So that** I can evaluate the API output without having to make a request myself.

## Background

The controller (story 005) already generates a `$kobold` array via `KoboldGeneratorService` and passes it to the view. This story adds the visual card in `home.blade.php` that renders that data as a formatted JSON block, giving visitors an immediate, concrete preview of the API output. Because it comes from `Cache::remember('homepage.kobold', 15, …)`, the grammar engine runs at most once per 15-second window.

## Acceptance Criteria

### Card structure

- [ ] A clearly headed section (e.g., `<h2>Sample Output</h2>` or equivalent) is present on the page.
- [ ] A brief sentence beneath the heading explains that the fields are generated by the Polygen grammar engine and change on each page load.
- [ ] The `$kobold` array is rendered using:
  ```blade
  <pre><code>{{ json_encode($kobold, JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE) }}</code></pre>
  ```
- [ ] The rendered JSON is contained in a styled card or highlighted block that is visually distinct from the surrounding page content.

### Data completeness

- [ ] All nine kobold fields are present in the rendered JSON output:
  `KoboldName`, `KoboldSurname`, `KoboldOrigin`, `KoboldColor`, `SpecialTrait`, `KoboldKinship`, `KoboldDish`, `KoboldStory`, `KoboldJob`.
- [ ] The rendered output is the live result from `KoboldGeneratorService`; it must not be hardcoded placeholder text.

### Styling

- [ ] The card is responsive and readable at 375 px (mobile) and 1280 px (desktop) viewport widths.
- [ ] Dark mode variants (`dark:` Tailwind classes) are applied so the card is legible in dark colour scheme.
- [ ] Long strings in the JSON do not break the card layout (use `overflow-x-auto` or `whitespace-pre-wrap` as appropriate).

## Out of Scope

- An interactive "regenerate" button that POSTs from the browser (Future Consideration in the spec).
- Any persistence or database changes.
- Syntax colouring of the JSON fields.

## Technical Notes

- The `$kobold` variable is always available in the view because the controller always passes it; no `@isset` guard is required, but a defensive `@if($kobold)` is acceptable.
- The cache key is `homepage.kobold` with a 15-second TTL — a page reload within that window will show the same kobold.
